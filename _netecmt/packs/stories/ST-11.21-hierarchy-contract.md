# User Story ST-11.21: Strategic Manifesto Hierarchy

**Persona:** Iuran (PM) / Athos (Arch)
**Status:** Ready

## 1. Descrição
Como arquiteto do sistema, preciso separar as entidades de **Funil (Estratégia)** e **Campanha (Execução)** para permitir que uma única estratégia de funil seja reaproveitada em múltiplas campanhas sazonais ou de teste A/B, garantindo a integridade dos dados de persistência do Darllyson (ST-11.15).

## 2. Critérios de Aceite
- [ ] Definição clara de `funnelId` (Blueprint Estratégico) vs `campaignId` (Manifesto Atômico).
- [ ] Atualização do contrato `campaign-handoff.md` para refletir a nova hierarquia.
- [ ] O objeto `CampaignContext` deve conter obrigatoriamente uma referência ao `funnelId` de origem.
- [ ] Garantir que o salvamento atômico utilize o `campaignId` como chave primária na coleção `campaigns`.

## 3. Justificativa Técnica (Athos)
Atualmente, o sistema usa `CampaignId == FunnelId`. Isso impede que o usuário crie uma nova campanha baseada no mesmo funil sem sobrescrever os dados da anterior. Ao separar os IDs, permitimos versionamento e histórico de execuções (Golden Thread).

## 4. Impacto em ST-11.15 (Darllyson)
O Darllyson precisará ler o `funnelId` para carregar a estratégia inicial, mas salvará todos os outputs (Copy, Design, Social) no documento identificado pelo `campaignId`.
