# Heurísticas: Arquitetura de Funil

## Contexto de Aplicação
Este documento apresenta regras práticas para estruturar funis de vendas usando o framework Value Ladder. Aplicável a qualquer negócio que venda produtos ou serviços online.

## 15 Regras de Arquitetura de Funil

### 1. Se você não tem um mapa claro de ofertas, ENTÃO você não tem um negócio escalável
**Porque**: Sem clareza, clientes não sabem para onde ir, e você não sabe como crescer.
**Exceção**: Negócios muito novos podem começar com uma oferta.
**Aplicação**: Mapear todas as ofertas atuais e futuras em uma escada de valor.

### 2. Se a primeira oferta é cara, ENTÃO você não vai ter clientes suficientes
**Porque**: Clientes precisam de confiança antes de gastar muito dinheiro.
**Exceção**: Clientes já conhecem você (marca estabelecida).
**Aplicação**: Começar com oferta de baixo risco (gratuita ou barata).

### 3. Se cada oferta não oferece mais valor que a anterior, ENTÃO clientes não vão subir a escada
**Porque**: Não há razão para gastar mais se não receber mais valor.
**Exceção**: Ofertas para segmentos diferentes (não sequencial).
**Aplicação**: Cada etapa deve ser claramente mais valiosa.

### 4. Se você não tem uma oferta recorrente, ENTÃO sua receita é imprevisível
**Porque**: One-time sales são voláteis; receita recorrente é estável.
**Exceção**: Negócios de projeto (consultoria de projeto único).
**Aplicação**: Incluir subscription ou membership em sua value ladder.

### 5. Se você trata todos os clientes igual, ENTÃO você deixa dinheiro na mesa
**Porque**: Diferentes clientes têm diferentes capacidades de pagamento.
**Exceção**: Mercado muito pequeno ou homogêneo.
**Aplicação**: Criar ofertas para diferentes faixas de renda.

### 6. Se você não começa simples, ENTÃO nunca vai lançar
**Porque**: Perfeição é inimiga do lançamento.
**Exceção**: Nenhuma - sempre começar simples.
**Aplicação**: Focar em um funil até $1M antes de adicionar complexidade.

### 7. Se sua macro value ladder é confusa, ENTÃO seus clientes vão ficar confusos
**Porque**: Clareza é responsabilidade do vendedor, não do cliente.
**Exceção**: Nenhuma - sempre deixar claro.
**Aplicação**: Documentar e simplificar sua value ladder regularmente.

### 8. Se você não tem um micro funil para cada oferta principal, ENTÃO está deixando conversão na mesa
**Porque**: Cada oferta merece sua própria sequência de vendas.
**Exceção**: Ofertas muito simples (one-step).
**Aplicação**: Criar funis específicos dentro da macro ladder.

### 9. Se a progressão de preço não faz sentido, ENTÃO clientes vão questionar
**Porque**: Preço deve refletir valor percebido.
**Exceção**: Promoções temporárias.
**Aplicação**: Aumentar preço conforme valor aumenta.

### 10. Se você não sabe onde seu cliente está na ladder, ENTÃO não sabe o que vender
**Porque**: Contexto é tudo em vendas.
**Exceção**: Nenhuma - sempre saber histórico de cliente.
**Aplicação**: Rastrear posição de cada cliente na value ladder.

### 11. Se não há transição clara entre ofertas, ENTÃO clientes vão cair
**Porque**: Transições precisam ser naturais e óbvias.
**Exceção**: Nenhuma - sempre ter transição clara.
**Aplicação**: Criar "ponte" entre cada etapa da ladder.

### 12. Se você não testa sua value ladder, ENTÃO não sabe se funciona
**Porque**: Suposições não substituem dados.
**Exceção**: Nenhuma - sempre testar.
**Aplicação**: A/B testar cada etapa da ladder.

### 13. Se sua oferta de topo é muito cara, ENTÃO poucos vão chegar lá
**Porque**: Nem todos têm capacidade de pagar o máximo.
**Exceção**: Mercado ultra-premium.
**Aplicação**: Oferta de topo deve ser acessível para top 10-20% de clientes.

### 14. Se você não comunica o valor em cada etapa, ENTÃO clientes não vão subir
**Porque**: Valor não é óbvio - precisa ser comunicado.
**Exceção**: Nenhuma - sempre comunicar valor.
**Aplicação**: Cada oferta deve ter proposta de valor clara.

### 15. Se sua value ladder não evolui com seu negócio, ENTÃO vai ficar obsoleta
**Porque**: Negócios mudam, ofertas precisam mudar.
**Exceção**: Nenhuma - sempre revisar e atualizar.
**Aplicação**: Revisar value ladder a cada trimestre.

## Checklist Rápido de Arquitetura

- [ ] Tenho mapa claro de todas as ofertas?
- [ ] Primeira oferta é de baixo risco/custo?
- [ ] Cada oferta é mais valiosa que a anterior?
- [ ] Tenho oferta recorrente?
- [ ] Ofertas atendem diferentes faixas de renda?
- [ ] Comecei simples?
- [ ] Macro ladder é clara?
- [ ] Cada oferta tem seu micro funil?
- [ ] Preços fazem sentido?
- [ ] Rastreio posição de cliente?
- [ ] Transições são claras?
- [ ] Testei cada etapa?
- [ ] Oferta de topo é acessível?
- [ ] Comunico valor em cada etapa?
- [ ] Reviso ladder regularmente?
