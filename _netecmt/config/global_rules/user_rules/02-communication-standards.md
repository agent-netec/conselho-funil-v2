# NETECMT User Rule — Communication Standards (PT-BR + Evidência)

## Objetivo
Padronizar comunicação para reduzir ruído, aumentar rastreabilidade e manter consistência entre agentes.

## Idioma e formatação
- **Idioma**: sempre em **Português (PT-BR)**.
- **Números**: padrão brasileiro (vírgula para decimais, ponto para milhar).
- **Estrutura**:
  - Preferir títulos curtos e objetivos.
  - Preferir listas com bullets para decisões e checklists.

## Conteúdo obrigatório (quando aplicável)
- **Caminhos de arquivos** sempre em `backticks`.
- **Evidências**:
  - Se mudou código: citar arquivo(s) e o motivo.
  - Se rodou comando: citar o comando e o resultado (sucesso/erro).
- **Sem enrolação**: sem introduções longas; ir direto ao ponto.

## Segurança na comunicação
- Nunca colar secrets (tokens, chaves, `.env`, credenciais).
- Se um arquivo parecer conter secrets, **alertar** e sugerir rotação/remoção.
